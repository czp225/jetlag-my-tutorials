Class HtmlRenderer

Renderer maintains a set of images, and can display them, along with text, onto the screen.

Renderer is responsible for managing the render loop, which effectively controls the whole game. However, due to asynchronous loading, initiating the render loop is tricky. First, we construct a renderer. Later, we call the loadAssets() function to get all the assets loaded. When loading is done, loadAssets will call its parameter, which can then initialize a world and start the render loop.

The operation of the render loop is also inverted. To render, first call initFrame(). Then use the add* methods to add things to the frame. Finally, call showFrame() and the frame will be rendered.
