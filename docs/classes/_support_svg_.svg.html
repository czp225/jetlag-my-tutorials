Class Svg

The Svg infrastructure allows the game designer to load SVG line drawings into a game. SVG line drawings can be made in InkScape. In JetLag, we do not use line drawings to their full potential. We only use them to define a set of lines for a simple, stationary obstacle. You should draw a picture on top of your line drawing, so that the player knows that there is an actor on the screen.

Private addLine

  • addLine(start: XY, stop: XY): void
    • Defined in support/Svg.ts:259

    This is a convenience method to separate the transformation and stretch logic from the logic for actually drawing lines

    There are two challenges. The first is that an SVG deals with pixels, whereas we like to draw actors in meters. This matters because user translations will be in meters, but SVG points and SVG translations will be in pixels.

    The second challenge is that SVGs appear to have a "down is minus" Y axis, whereas our system has a "down is plus" Y axis. To get around this, we reflect every Y coordinate over the horizontal line that intersects with the first point drawn.

    Parameters

    • start: XY

      The point from which the line originates

    • stop: XY

      The point to which the line extends

    Returns void

Private processD

  • processD(d: string, readonlymode: boolean): void
    • Defined in support/Svg.ts:138

    The root of an SVG drawing will have a g element, which will have some number of path elements. Each path will have a "d=" attribute, which stores the points and information about how to connect them. The "d" is a single string, which we parse in this function.

    Parameters

    • d: string

      The string that describes the path

    • readonlymode: boolean

      Are we in read-only mode (true), where we are computing the top/left pixel boound, or are we in draw mode (false), where we actually draw the lines

processFile

  • processFile(file: string, x: number, y: number, stretchX: number, stretchY: number, world: WorldApi, cfg: JetLagConfig, cb: function): void
    • Defined in support/Svg.ts:83

    Process an SVG file and go through all of the "path" elements in the file. For each path, create a thin obstacle, and then run the provided callback on the obstacle.

    Note that we ignore SVG /translate/ directives... we let the programmer translate the drawing instead.
